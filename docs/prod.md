# 当前游戏功能文档（MVP 现状）

> 目标：记录当前版本“已经可用”的游戏功能，后续需求统一在此文档中持续归档与扩展。

## 1. 项目定位与技术栈
- 类型：单机回合制卡牌原型（Phaser + TypeScript + Vite）。
- 运行方式：浏览器运行，`Phaser.Game` 挂载到页面 `#app`。
- 当前定位：玩法链路与界面均为 MVP，占位内容较多，但核心循环已打通。

## 2. 当前可用场景与流程

### 2.1 场景列表
- `BootScene`：启动后立即跳转角色选择。
- `CharacterSelectScene`：展示角色信息、初始卡组与“开始修行”按钮。
- `CultivationScene`：修仙界主界面，展示角色基础属性、卡组按钮、修炼卡牌区。
- `EventScene`：事件流程占位页，支持事件步数递增（1/20 ~ 20/20）。
- `BattleScene`：战斗玩法场景（与修仙界/事件链路目前部分解耦，见“已知限制”）。

### 2.2 当前默认主流程
1. 启动游戏后进入角色选择。
2. 点击“开始修行”进入修仙界场景（`CultivationScene`），并传入 `characterId`。
3. 在修仙界中可点击修炼卡牌；打出「游历山河」后进入事件场景。
4. 在事件场景中点击选项 A/B 推进步数。
5. 当步数超过 20 时，回到角色选择场景。

## 3. 角色与基础数据

### 3.1 角色系统（当前仅 1 名）
- 角色：`swordsman`（剑修）
- 境界：炼气
- 最大生命：120
- 初始灵力：3

### 3.2 初始卡组
- 挥砍 x5
- 招架 x5
- 还击（`riposte`）x1


## 4. 修仙界系统（CultivationScene）
- 展示基础属性：境界、境界进度、血量、符箓槽位、丹药槽位、精力、寿命、道心、正邪值、灵石。
- 提供“修炼卡组”按钮，点击可查看当前修炼卡组内容。
- 提供“战斗卡组”按钮，点击可查看角色初始战斗卡组。
- 修炼卡牌区包含初始卡牌：坊市x1、引气入体x5、参悟功法x1、游历山河x1、问心x1。
- 打出「游历山河」会跳转到 `EventScene`；其余卡牌暂为占位效果提示。

## 5. 事件系统（EventScene）
- 使用 `registry` 保存当前角色对象（由 `characterId` 查询）。
- 页面显示境界、当前事件步数、占位事件标题。
- 提供两个可点击选项（删牌 / 升级）作为占位行为。
- 每次点击任一选项：`step + 1`。
- `step > 20`：返回角色选择（未来可替换为 Boss 战或结算节点）。

## 6. 战斗系统（BattleScene）

### 6.1 战斗基础规则
- 玩家初始：`hp=70`、`energy=3`、`block=0`。
- 敌人：两只狼（不同生命与意图伤害）。
- 回合开始：重置灵力与格挡，抽到 5 张手牌。
- 回合结束：弃掉手牌，进入敌方行动。

### 6.2 牌堆与手牌
- 包含 `drawPile / discardPile / hand` 三类容器。
- 抽牌堆空时：若弃牌堆有牌则洗回抽牌堆。
- 手牌点击逻辑：
  - 攻击牌：进入选目标模式（点击敌人确认目标）。
  - 非攻击牌：直接结算。

### 6.3 目标选择与交互
- 攻击牌使用时显示提示文本与跟随鼠标的箭头。
- 可通过 `Esc` 或右键取消选目标。
- 选目标阶段会临时启用敌人交互并高亮提示。

### 6.4 已接入的卡牌效果（当前场景内最小实现）
- 挥砍：对单体造成 6 伤害。
- 招架：获得 5 格挡。

> 说明：仓库中已存在更完整的卡牌数据与结算模块（如 `cardEffects.ts`、`data/cards.ts`），但当前 `BattleScene` 仍以 MVP 逻辑为主，尚未完全切换为统一结算管线。

### 6.5 敌人与胜负
- 敌方回合：存活敌人依次对玩家造成意图伤害，然后随机下回合意图。
- 玩家生命归零：延迟返回角色选择。
- 敌方全灭：提示胜利并延迟跳转事件场景（传固定角色 id）。

## 7. UI 与体验能力
- 响应式缩放：`Phaser.Scale.FIT + CENTER_BOTH`。
- 支持全屏切换按钮（角色选择 / 战斗场景内）。
- 战斗中有基础动画反馈：受击闪烁、轻微位移、悬停上浮等。
- 日志文本区域可显示战斗过程信息。

## 8. 已知限制（建议优先归档为后续需求）
1. **流程链路未完全闭环**：事件页尚未进入战斗；战斗胜利后回事件是独立跳转。
2. **角色属性未贯通战斗**：战斗场景当前玩家数值未读取角色配置。
3. **卡牌系统存在“双轨”**：`BattleScene` 内部与 `data/cards.ts + cardEffects.ts` 并行，建议统一。
4. **数据结构命名不一致**：角色初始卡组里展示名“还击”，卡牌 id 对应 `riposte/雁回`，需统一命名策略。
5. **内容仍占位**：事件选项、成长、奖励、地图、Boss 等尚未实装。
