# 当前游戏功能文档（以代码为准）

> 目标：记录当前版本“已经可用”的功能与已知限制，便于后续迭代对齐。

## 1. 项目定位与技术栈
- 类型：单机回合制卡牌原型。
- 技术：Phaser + TypeScript + Vite。
- 运行方式：浏览器端运行，`Phaser.Game` 挂载到 `#app`。

## 2. 场景与入口

### 2.1 已注册场景（`main.ts`）
- `BootScene`
- `CharacterSelectScene`
- `CultivationScene`
- `EventScene`
- `BattleScene`

### 2.2 当前默认流程
1. 启动后进入 `BootScene`，立即跳转 `CharacterSelectScene`。
2. 在角色选择页点击“开始修行”，进入 `CultivationScene`。
3. 在修真界可进行周期卡拖拽与轮次触发（当前为占位数值与提示逻辑）。

> 说明：`EventScene` 与 `BattleScene` 已实现，但默认主流程中尚未从修真界直接串联进去。

## 3. 角色与基础数据

### 3.1 角色（当前仅 1 名）
- 角色：`swordsman`（剑修）
- 境界：炼气
- 最大生命：120
- 初始灵力（战斗资源字段）：3

### 3.2 角色初始战斗卡组
- 挥砍 x5
- 招架 x5
- 还击（id: `riposte`）x1

## 4. 角色选择（CharacterSelectScene）
- 展示角色名、初始境界、生命/灵力、初始卡组。
- 提供“开始修行”按钮，进入 `CultivationScene`。
- 提供全屏切换按钮。

## 5. 修真界（CultivationScene）

### 5.1 人物面板（当前显示）
- 左侧：境界、当前血量、符箓槽位、丹药槽位、寿元、道心、灵石。
- 右侧：精力 `3/3`、灵力 `0`、神识 `0`、道韵 `0`。
- 周期状态文本：`准备阶段 / 阶段一 / 阶段二 / 阶段三 / 阶段四`。
- 已移除“境界进度”显示。

### 5.2 顶部按钮
- 周期卡组：toast 显示当前周期卡构成。
- 修炼卡组：toast 提示“敬请期待”。
- 战斗卡组：toast 显示当前角色初始战斗卡组。

### 5.3 周期卡系统（当前主玩法）
- 初始周期卡池：
  - 坊市 x1
  - 游历山河 x1
  - 历练 x2
  - 悬赏 x1
- 玩法：
  - 底部卡牌可拖拽到 4 个轮次槽位（每槽 1 张）。
  - 槽位卡牌可通过 `×` 删除并回到原位。
  - 点击“开始修行”后按轮次触发槽位卡（toast 提示），槽位底色会变化。
  - 完成 4 轮后触发“进入下一周期（占位）”提示并重置轮次状态。

### 5.4 当前限制
- 周期卡触发结果目前主要是 UI/提示层面的占位逻辑。
- “游历山河”目前不会在修真界中直接切场景到事件页。

## 6. 事件（EventScene）
- 页面展示：境界、当前事件步数（`1/20` 起）、占位事件标题。
- 选项：A（删牌占位）/ B（升级占位）。
- 规则：每次点击任一选项 `step + 1`。
- 当 `step > 20`：返回角色选择页。

## 7. 战斗（BattleScene）

### 7.1 基础规则
- 玩家初始：`hp=70`、`energy=3`、`block=0`。
- 敌人：两只狼（不同生命与意图伤害）。
- 玩家回合开始：重置灵力和格挡，补至 5 张手牌。
- 玩家结束回合：弃手牌，进入敌方回合。

### 7.2 卡牌与结算（当前 BattleScene 内置）
- 当前战斗内置卡：
  - 挥砍：造成 6 伤害（攻击牌，需选敌人）。
  - 招架：获得 5 格挡（技能牌，无目标）。
- 抽牌堆/弃牌堆/手牌完整可用，抽空会洗回弃牌堆。
- 支持选目标模式、Esc/右键取消、箭头跟随提示。

### 7.3 胜负与去向
- 玩家生命归零：延迟回到角色选择。
- 敌方全灭：提示胜利并延迟跳转 `EventScene`（固定 `characterId: swordsman`）。

## 8. UI 与交互能力
- 全局缩放：`Phaser.Scale.FIT + CENTER_BOTH`。
- 全屏按钮：角色选择、修真界、战斗场景均支持。
- 战斗场景包含基础动画与日志区（受击/悬停/文本日志）。

## 9. 已知限制（当前版本）
1. 主流程尚未闭环：修真界未直接进入事件或战斗。
2. 事件与战斗存在，但与修真界的联动仍是占位状态。
3. 战斗数值仍未读取角色实时养成属性（目前使用战斗场景内置初始值）。
4. 战斗卡牌体系与 `data/cards.ts`、`cardEffects.ts` 尚未完全统一。
5. 文案命名存在历史差异：`starterDeck` 展示“还击”，`data/cards.ts` 中 `riposte` 名称为“雁回”。

## 10. 快速启动
```bash
npm install
npm run dev
```

生产构建：
```bash
npm run build
```
